## 需求概述

对现有武术减脂训练看板进行两项重大改造：浅色主题切换 + 6个新功能模块扩展。

## 用户要求

### 1. 浅色背景

将当前深色 Cyberpunk 风格页面改为浅色背景主题

### 2. 新增6个功能模块

- 打卡日历：月度日历视图，打卡/缺勤一目了然
- 数据仪表盘：训练频率、体重趋势、减脂进度图表
- 体重追踪：每日体重记录 + 趋势曲线 + BMI 自动计算
- 饮食记录：每日三餐 + 加餐记录，简单热量估算
- 成就系统：连续打卡勋章、阶段目标达成、里程碑奖励
- 数据管理：JSON 导出/导入备份、清空数据

## 用户画像

偶尔锻炼、有一定运动基础的用户，需要移动端友好的可视化训练管理工具。

## 技术方案

### 整体策略

在现有单文件 HTML 基础上完成所有改动，保持自包含、无外部依赖的架构。由于改动涉及 CSS 变量全量替换、Tab 系统重构、以及 6 个新模块的 HTML+CSS+JS 实现，文件改动量较大（预计从 1677 行扩展到 ~4500 行），采取**整体重写单文件**策略，确保一致性。

### 实现方案

#### A. 浅色主题改造

修改 `:root` CSS 变量（第 10-31 行）：

- 背景色：`--bg-primary: #F5F7FA`, `--bg-secondary: #EBEDF2`, `--bg-card: #FFFFFF`, `--bg-card-hover: #F8F9FB`
- 功能色降饱和度适配浅底：`--neon-green: #10B981`, `--neon-blue: #3B82F6`, `--neon-orange: #F59E0B`, `--neon-red: #EF4444`, `--neon-yellow: #FBBF24`
- 文字色反转：`--text-primary: #1A202C`, `--text-secondary: #4A5568`, `--text-muted: #A0AEC0`
- 玻璃态适配：`--glass-bg: rgba(255,255,255,0.75)`, `--glass-border: rgba(0,0,0,0.06)`
- 辉光阴影改为柔和投影：`--shadow-glow-green: 0 2px 12px rgba(16,185,129,0.12)`

同步修改 body::before 背景渐变（第 50-63 行），降低浓度，以及散布在 CSS 中的硬编码 `rgba` 值（约 20 处）。

#### B. Tab 系统重构（4 → 8 Tab）

- `.tab-nav` 改为横向可滚动：移除 `flex:1`，添加 `overflow-x: auto`, `gap: 4px`
- `.tab-btn` 改为 `flex-shrink: 0; white-space: nowrap; padding: 10px 16px`
- Tab 列表：训练计划、打卡日历、体重追踪、饮食记录、数据仪表盘、成就系统、饮食方案、训练须知、数据管理
- 将"饮食方案"和"训练须知"（纯内容）置于 Tab 尾部，新功能 Tab 居前

#### C. 新增功能模块

**打卡日历**

- 月度网格：7列（周一~周日），动态渲染当月天数
- 左右箭头切换月份，居中显示"YYYY年MM月"
- 日期状态：已打卡（绿色圆点）、缺勤（红色圆点，仅已过训练日）、今日（蓝色边框）、未来/休息（灰色）
- 与现有 `martial_plan_done` 数据联动，映射日期到计划中的训练日
- 底部当月统计栏

**体重追踪**

- 输入表单：体重(kg)、日期、身高(cm，首次输入后缓存)
- BMI 实时计算：`weight / (height/100)^2`，按中国标准分类
- Canvas 2D 趋势曲线：带网格线、数据点、平滑连线
- 历史记录列表，支持单条删除
- localStorage key: `martial_weight` (数组: `[{date, weight}]`), `martial_profile` (对象: `{height}`)

**饮食记录**

- 日期选择器（前后切换）
- 四餐卡片：早餐/午餐/晚餐/加餐，每餐可添加多条"食物名称+热量"
- 内嵌简化食物热量库（约 40 种常见食物，分类展示便于选择）
- 当日总热量汇总 + 目标热量(2000kcal)对比进度条
- localStorage key: `martial_diet` (对象: `{date: {breakfast:[],lunch:[],dinner:[],snack:[]}}`)

**数据仪表盘**

- 训练频率：本周完成率（环形进度 CSS），本月训练天数
- 体重趋势：复用 Canvas 绘制体重曲线（与体重追踪共享绘制逻辑）
- 减脂进度：总消耗千卡、预估减脂量（7700kcal ≈ 1kg脂肪）、训练天数/40天

**成就系统**

- 成就定义数组（约 12 个）：首次打卡、连续3/7/14/30天、累计10/20/30/40天、体重记录首次、减重2/5kg
- 每个成就：图标+名称+描述+达成条件+达成日期
- 已达成：彩色卡片 + 解锁时间；未达成：灰色 + 锁图标
- 自动检测函数：每次打卡/记录体重时触发
- localStorage key: `martial_achievements` (对象: `{achievementId: unlockDate}`)

**数据管理**

- 导出 JSON：收集所有 localStorage key 打包为 JSON 文件下载
- 导入 JSON：文件选择器读取，解析后写入各 localStorage key，刷新页面
- 清空数据：红色警告按钮，点击后弹出自定义确认弹窗（非 confirm()），二次确认后清空
- 显示当前数据量统计

#### D. 数据存储架构

```
martial_plan_done     (已有)  {week-day: {date, calories}}
martial_weight        (新增)  [{date:"2026-04-03", weight:70.5}, ...]
martial_profile       (新增)  {height:170}
martial_diet          (新增)  {"2026-04-03": {breakfast:[{name,cal}],...}, ...}
martial_achievements  (新增)  {"first_checkin":"2026-04-03", ...}
```

#### E. Canvas 图表绘制

- 抽取公共绘制工具函数：`drawLineChart(canvas, data, options)`
- 支持配置：颜色、网格、标签、Y轴范围、数据点样式
- 体重趋势和仪表盘共用此函数

### 性能与可靠性

- Canvas 图表按需绘制（Tab 切换到时才渲染），避免不可见时浪费计算
- localStorage 操作均用 try-catch 包裹，防止隐私模式/存储满时崩溃
- 食物热量库为静态 JS 数组，不占用存储空间

### 爆炸半径控制

- 训练计划 Tab 的数据和渲染逻辑保持不变
- 饮食方案和训练须知 Tab 的 HTML 内容保持不变
- 打卡功能（handleCheckin）保持不变，仅增加成就检测触发

## 设计风格

浅色清新运动风。以白色和浅灰为基调，搭配柔和的功能色点缀，营造清爽、专业、有活力的运动应用视觉感受。保留卡片式布局和微交互动画，但去除深色赛博朋克元素，改为更加明快的设计语言。

## 页面规划

### Tab 导航栏

- 横向可滑动的 Tab 栏，9个 Tab 按钮等宽排列，超出部分可左右滑动
- 当前选中 Tab 带底部彩色指示条 + 浅色背景高亮
- 置顶吸附 (sticky)，背景半透明模糊

### 训练计划页（保留原有布局）

- Block 1：统计卡片行（已完成/连续打卡/消耗千卡），浅底白卡片，功能色数字
- Block 2：整体进度条（绿色渐变填充）
- Block 3：周次选择器（横向滑动）
- Block 4：7天日历卡片
- Block 5：训练详情（四阶段卡片）
- Block 6：打卡按钮

### 打卡日历页

- Block 1：月份导航栏（左箭头 + 月份标题 + 右箭头）
- Block 2：星期标题行（周一~周日）
- Block 3：月度日历网格（6行7列，日期圆点颜色编码）
- Block 4：当月统计栏（训练X天/缺勤X天/出勤率X%）

### 体重追踪页

- Block 1：BMI 卡片（大字 BMI 值 + 分类标签 + 身高输入）
- Block 2：体重趋势 Canvas 图表
- Block 3：体重记录输入表单（体重+日期+添加按钮）
- Block 4：历史记录列表（可删除）

### 饮食记录页

- Block 1：日期切换导航
- Block 2：当日热量总览（已摄入/目标 + 进度条）
- Block 3：四餐卡片（早餐/午餐/晚餐/加餐），每餐可展开添加食物
- Block 4：快速添加（常见食物热量库快捷选择）

### 数据仪表盘页

- Block 1：本周训练频率环形进度
- Block 2：体重趋势迷你图
- Block 3：减脂数据总览（总消耗/预估减重/完成率）

- Block 1：成就统计（已解锁X/总数Y）

### 数据管理页
