## 需求概述

将现有"饮食记录"Tab删除，替换为基于体质分析的个性化精准饮食推荐系统。

### 用户意图

- 饮食记录功能手动输入每餐热量不准确，体验差，需要删除
- 希望根据个人体质数据（性别、年龄、身高、体重、活动水平）进行精准营养推荐
- 生成个性化饮食方案，训练日与休息日区分

### 功能要求

1. 删除"饮食记录"Tab（含食物热量库、手动记录功能）
2. 保留"饮食方案"Tab，改造为基于体质数据的动态精准推荐
3. 新增体质评估表单（性别、年龄、身高、体重、目标体重、活动水平）
4. 基于Mifflin-St Jeor公式计算BMR和TDEE
5. 根据体质类型（偏胖/偏瘦/正常）生成差异化营养素分配
6. 训练日/休息日分别给出热量目标和营养素配比
7. 生成具体的一日食谱推荐（根据计算出的热量目标）

### 影响范围

- Tab导航从9个减为8个
- 仪表盘中的"饮食记录(天)"统计需清理
- 数据导出导入中的diet数据相关代码需清理
- profile存储对象需扩展
- 清理FOOD_LIB、MEALS等不再使用的常量

## 技术方案

### 实现策略

在现有单文件HTML基础上进行定向修改，通过删除+替换的方式完成改造，保持其他Tab功能不受影响。

### 修改清单

**A. 删除清理（约6处）**

1. **Tab导航**（行860）：删除 `<button class="tab-btn" data-tab="diet-record">饮食记录</button>`
2. **饮食记录Tab HTML**（行926-942）：删除整个 `<div class="tab-content" id="tab-diet-record">...</div>` 块
3. **饮食记录CSS**（行651-700区域）：删除 `.food-lib-toggle`、`.food-lib`、`.food-cat`、`.food-grid`、`.food-chip`、`.meal-section`、`.meal-section-header`、`.meal-section-body`、`.meal-add-row`、`.diet-item`、`.diet-calorie-overview` 相关样式
4. **饮食记录JS**（行1682-1780）：删除 `let dietDate=todayStr()`、`MEALS`、`CAL_TARGET`、`renderDietRecord()`、`renderFoodLib()`、以及对应的 `foodLibToggle`、`dietPrev`、`dietNext` 事件监听
5. **FOOD_LIB常量**（行1303-1330）：删除整个食物热量库对象
6. **状态管理和导出导入**：删除 `dietData` 变量及其 `loadData`/`saveData` 调用；仪表盘中删除 `dietCount` 引用；数据管理Tab的导出导入中删除 `dietData` 相关代码；清空数据中删除 `diet` key

**B. 改造"饮食方案"Tab为精准推荐（行998-1047）**

将现有静态HTML替换为动态渲染区域 `<div id="dietPlanContent"></div>`，由JS函数根据profile数据动态生成内容。

**C. 扩展profile存储**

现有 `profile` 对象从 `{height:170}` 扩展为：

```
{height, gender:'male'|'female', age:25, activityLevel:1-5, goalWeight:65}
```

存储在 `martial_profile`，所有新增字段使用 `||` 默认值保证向后兼容。

**D. 新增JS逻辑**

1. **体质评估表单渲染**：在饮食方案Tab顶部渲染表单，首次使用时显示引导卡片
2. **TDEE计算**：

- BMR（Mifflin-St Jeor）：男=10*体重+6.25*身高-5*年龄+5，女=10*体重+6.25*身高-5*年龄-161
- 活动系数：久坐1.2/轻度1.375/中度1.55/高度1.725/极高1.9（结合武术训练取1.55-1.725）
- TDEE = BMR * 活动系数

3. **营养素分配**：根据BMI分类差异化

- 偏胖（BMI>=24）：蛋白35% / 碳水35% / 脂肪30%
- 正常（18.5<=BMI<24）：蛋白30% / 碳水40% / 脂肪30%
- 偏瘦（BMI<18.5）：蛋白25% / 碳水45% / 脂肪30%

4. **训练日/休息日热量**：

- 训练日：TDEE - 400（温和缺口）
- 休息日：TDEE - 200（小缺口，保障恢复）

5. **食谱推荐生成**：根据计算出的热量目标和营养素比例，从预设食谱模板库中匹配推荐
6. **体质类型检测**：根据BMI + 腰围(可选)判断偏胖/正常/偏瘦

**E. 食谱模板库**

预设3套食谱（偏胖/正常/偏瘦），每套包含训练日和休息日的三餐+加餐，热量按比例缩放。比现有静态内容更丰富（5种以上可选搭配）。

### 性能与可靠性

- TDEE计算在前端即时完成，无需API调用
- 表单数据实时保存localStorage，刷新不丢失
- 体质数据变化后自动重新计算并更新推荐

### 爆炸半径控制

- 训练计划、打卡日历、体重追踪、成就系统、训练须知、数据管理Tab的HTML和JS逻辑完全不动
- 体重追踪中的profile.height使用保持兼容（扩展而非替换）
- 仪表盘仅清理dietCount引用，其余逻辑不变

## 设计方案

将"饮食方案"Tab改造为精准体质推荐系统，包含体质表单和数据展示两大区域。

### 页面布局

**Block 1：体质档案卡片**

- 顶部大卡片，包含表单输入区：性别（按钮组男/女）、年龄（数字）、身高（数字）、体重（数字，与体重追踪联动）、目标体重（数字）、活动水平（5档下拉选择）
- 保存按钮，数据存入profile
- 已填写时显示BMI标签和体质类型标签

**Block 2：热量与营养概览**

- 4格统计卡：基础代谢BMR、日总消耗TDEE、训练日目标热量、休息日目标热量
- 营养素分配条形图（蛋白/碳水/脂肪比例可视化）

**Block 3：训练日食谱推荐**

- 标题"训练日推荐方案"
- 早餐/午餐/训练后加餐/晚餐四个卡片
- 每个卡片包含2-3个可选搭配（A/B选项）
- 每餐标注热量

**Block 4：休息日食谱推荐**

- 标题"休息日推荐方案"
- 早餐/午餐/晚餐三个卡片
- 结构同上

**Block 5：减脂饮食核心原则**

- 保留现有原则内容，根据体质类型微调建议

**Block 6：减脂期避免食物**

- 保留现有内容

### 交互设计

- 表单字段变化时实时计算BMR/TDEE并更新展示
- 体质类型变化时自动切换食谱模板
- 首次进入显示引导提示"填写体质信息获得个性化推荐"
